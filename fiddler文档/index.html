一、Fiddler 介绍

Fiddler 是一个通过代理来进行 http 抓包的工具。与 wireshark 相比，它轻量简单易上手，专门用于 http(s) 数据包的抓取。除了记录 http 包外，Fiddler 还能做到设置断点、修改请求和响应数据以及模拟弱网络环境。另外还能安装插件对功能进行扩展，甚至编写脚本实现一些自动化操作。

运行时会在本地建立一个代理服务，默认地址：127.0.0.1:8888。Fiddler 开启后配置本机代理，IE 浏览器的 proxy 会自动变成 127.0.0.1:8888，借此可以抓取 IE 浏览器与服务器之间的所有 HTTP(s) 请求，而火狐、chrome 则需要手动设置代理。

如果系统中没有设置代理，即采用直连网络，那么当用户浏览网页时，浏览器会通过 tcp 连接向服务器发起 http 请求，服务器在收到请求后进行响应。

而如果系统设置了静态 http 代理，那么浏览器发送的所有 http(s) 请求都会被重定向到代理服务器，代理服务器会根据请求的目的 ip 地址再将请求转发到相应服务器。同理，服务器返回给浏览器的数据也要经过代理这一层。

Fiddler 就相当于代理服务器。当启用 Fiddler 后，默认会将 IE 浏览器的 http(s) 代理设置为 127.0.0.1:8888（火狐、chrome 则需要手动设置代理）。 Fiddler 会在本地8888端口进行监听， 从而拦截所有 http(s) 会话。Fiddler 监听的端口号可更改。

使用http://localhost:8888/可以访问到Fiddler的echo服务

在了解Fiddler解密https的原理之前，我们首先要知道标准的https通信原理。在https通信中，运用到了两种加密技术——对称加密技术和非对称加密技术（RSA），非对称加密用来在握手过程中传输对称秘钥，而对称加密技术是握手完成之后实际使用的通信加密技术。

https实际通信过程可以分为两步走：

第一步，客服端和服务器协商通信使用的密钥（对称密钥）。

这个过程可以简单描述为——客户端发起请求获取服务器的证书，证书里包含了服务器的RSA公钥。客户端生成通信用的对称密钥，使用服务器的公钥加密后传输给服务器。服务器接收到之后使用自己的RSA私钥解密得到传输用的明文秘钥。

第二步，客服端和服务器使用协商好的密钥进行加密通信。
